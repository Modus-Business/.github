## Modus는 무엇인가요?

Modus는 학생들이 모둠 과제와 토론을 진행할 때 더 건강하게 협업할 수 있도록 돕는 서비스입니다.

AI가 학생을 평가하거나 대화를 통제하는 것이 아니라, 학생들이 직접 대화하고 결정하는 흐름을 유지하면서 필요한 순간에만 표현을 부드럽게 다듬고, 참여를 자연스럽게 열어주며, 논의를 더 깊게 만드는 역할을 합니다.

---

## 우리가 해결하려는 문제

모둠 활동은 많은 수업에서 사용되지만, 실제 현장에서는 다음과 같은 문제가 자주 발생합니다.

| 문제 | Modus의 접근 |
| --- | --- |
| 감정적이거나 거친 표현 | 전송 전에 부드러운 표현을 제안합니다. |
| 일부 학생에게 대화가 집중됨 | 전체 그룹을 향해 자연스럽게 참여를 유도합니다. |
| 말수가 적은 학생이 소외됨 | 특정 학생을 지목하지 않고 열린 질문으로 참여 기회를 만듭니다. |
| 기여도를 메시지 수로만 판단함 | 질문, 정리, 자료 제안, 갈등 완화 등 역할 중심으로 바라봅니다. |
| 익명성과 책임감의 균형이 어려움 | 설문 기반 닉네임으로 부담은 낮추고 개성은 살립니다. |

## Modus가 제공하는 경험

### 설문 기반 닉네임

사용자의 협업 성향과 의사소통 방식을 반영해 한국어 닉네임을 제안합니다.

예시:

```text
차분한 설계자
부드러운 조율자
조용한 해결사
```

닉네임은 성별, 외모, 능력, 민감한 개인정보를 드러내지 않도록 설계합니다.

---

### 채팅 전송 전 문장 조언

학생이 메시지를 보내기 전에 AI가 표현을 점검합니다.

공격적이거나 상처가 될 수 있는 문장은 바로 차단하기보다, 먼저 부드러운 수정안을 제안합니다.

예시:

```text
이 표현은 조금 강하게 들릴 수 있어요.
조금 더 부드럽게 표현해볼까요?
의견은 유지하되 표현만 완화해보는 건 어떨까요?
```

---

### 그룹 대화 참여 유도

대화가 특정 학생에게만 집중되거나 참여가 줄어드는 흐름을 감지하면, AI가 전체 그룹을 향한 중립적인 문장을 제안합니다.

예시:

```text
다른 의견도 편하게 말해볼까요?
아직 나오지 않은 관점도 있을 것 같아요.
각자 한 가지씩 아이디어를 던져볼까요?
```

특정 학생을 공개적으로 지목하지 않습니다.

---

### 토론 심화 질문

AI는 정답을 알려주는 대신, 학생들이 더 깊게 생각할 수 있는 질문을 제안합니다.

예시:

```text
이 선택을 사용자 관점에서 보면 어떤 장단점이 있을까요?
반대 입장에서 보면 어떤 문제가 생길 수 있을까요?
지금까지 나온 의견을 정리하면 선택지는 몇 가지인가요?
```

---

### 역할 기반 기여 분석

Modus는 단순히 말을 많이 한 학생을 높게 평가하지 않습니다.

다음과 같은 협업 역할을 함께 봅니다.

```text
논의 시작
질문 제기
답변 제공
자료 제안
요약 및 정리
갈등 완화
실행 기여
의사결정 기여
```

기여 분석은 학생을 낙인찍기 위한 점수가 아니라, 교사와 운영자가 더 나은 피드백을 제공하기 위한 보조 지표입니다.

## AI 설계 원칙

Modus의 AI는 다음 역할을 지향합니다.

| 하는 역할 | 하지 않는 역할 |
| --- | --- |
| 협업 도우미 | 정답 대필 |
| 대화 촉진자 | 학생 평가 확정 |
| 표현 완화 도우미 | 사용자 낙인찍기 |
| 사고 확장 보조자 | 공개적으로 비교하기 |
| 정리 및 요약 보조자 | 대화 주도권 빼앗기 |

---

## 우리가 중요하게 생각하는 것

- 학생을 공개적으로 지적하지 않는 AI
- 차단보다 수정 제안을 우선하는 경험
- 참여가 적은 학생을 직접 지목하지 않는 개입
- 메시지 수보다 역할과 맥락을 보는 기여 분석
- 익명성과 책임감을 함께 고려한 닉네임
- 수업 흐름을 방해하지 않는 절제된 AI 개입

---

## Modus가 만들고 싶은 협업

Modus는 완벽한 답을 대신 만들어주는 서비스가 아닙니다.

우리는 학생들이 서로의 생각을 더 안전하게 꺼내고, 더 균형 있게 참여하고, 더 깊게 토론할 수 있는 협업 환경을 만들고자 합니다.
